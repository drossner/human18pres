What does Spatial Hypertext mean, in context of EUD tools?

User can use given space to position his modules (right). On the left side, position is fixed.

Well, that's great so far, but

  • the application should interpret the spatial structure like a human would do
  • and use this information to improve the process of building an application (or whatever).

How spatial information (may) improve user experience of EUD tools

Planning and designing of "code"

  • Laypersons have limited knowledge about the tool they use
  • Fast changing requirements
  • Modifications must be possible, without the need of reprogramming whole parts

From a rough idea to a functional application

  • "What do I want to achieve?"
  • "Which components can help me?"
  • "Is there a good place to experiment?"
  • Context sensitive hints/suggestions
  • Context is a computer generated interpretation of the workspace,
  • that is as close as possible to that of a human
  • Of course, a suitable knowledge base is needed...

Example case: HEIMDALL

  • Part of ODIN
  • Open Data Innovation
  • Platform for professional developers and laypersons to work with open data
  • Developers create components (traditional coding)
  • End users are working with a graphical tool to compose these components
  • A journalist (e.g.) wants to aggregate weather data with traffic accidents
  • Takes two data sources (weather and traffic data)
  • Combine them with a aggregation/filter/ ... component
  • Adds a visualization component
